# Epi-Logos Coordinate Map

## Status

This map is a harmonised bridge from the older `Idea/Empty` coordinate files into current canon. It should help agents orient quickly, but it no longer claims to be the actively maintained source of truth for S/S′ implementation state.

Use current authorities in this order:

1. [[World-Ontology]] for the stable ontology hub.
2. [[S-SYSTEM-INDEX]] for the current S/S′ spine.
3. [[S0-SPEC]], [[S1-SPEC]], [[S2-SPEC]], [[S3-SPEC]], [[S4-SPEC]], and [[S5-SPEC]] for level detail.
4. [[ARCHITECTURE-DIAGRAM-PACK]] for architecture diagrams and visual topology.

[[Idea/Empty]] remains a temporal/working surface. Durable coordinate definitions should graduate into [[World]] or [[World/Types]] through [[Hen]].

## Level Notes

### S0 / S0′ — Terminal And Command Membrane

**Current role:** [[S0]] makes the system executable. It owns shell/process execution, CLI commands, local env/tool facts, command topology, and test harnesses.

**Prime law:** [[S0′]] is the typed command membrane: preferred tools, runtime surface, portal command registry, launch/session/tmux/cmutex orchestration, and coordinate-native command validation.

**Original definition worth preserving:** S0 is the ground where computation meets command. The old "filesystem ground / atomic commands / stream transformation / shell context / package integration" sub-coordinate intuition remains strong, but belongs under command-membrane law rather than a separate QL type-enforcement API.

**Current gap:** keep routing product-native aliases back to coordinate-native S0 methods; continue extracting command law from older root adapters into Body/S authorities.

### S1 / S1′ — Vault, Hen, CT, And Residency

**Current role:** [[S1]] makes artifacts resident. It owns [[Idea]] vault structure, file reads/writes, frontmatter, wikilinks, template surfaces, and architecture-as-documentation residency.

**Prime law:** [[S1′]] is [[Hen]]: CT templates, folder/MOC/canvas law, `c_0_source_coordinates`, frontmatter validation, wikilink topology, residency resolution, and graduation from [[World/Types]] into flat [[World]].

**Original definition worth preserving:** each file as a monad remains useful: perception = source links, appetite = outgoing links/intent, pre-established harmony = coordinate/wikilink/graph sync. This should inform Hen workflow language without becoming a parallel "Monadological Mapping API" root.

**Current gap:** public Hen surfaces such as `s1'.world.graduate`, `s1'.world.link_infer`, `s1'.moc.ensure`, and `s1'.graph.intent.world_types` are specified but not fully exposed as stable gateway/tool methods.

### S2 / S2′ — Graph, Retrieval, And Schema Validation

**Current role:** [[S2]] gives the system graph, retrieval, semantic cache, and coordinate property law.

**Prime law:** [[S2′]] validates graph labels/properties/relationships, resolves coordinate-aware retrieval, and turns vault/path/wikilink evidence into graph state only through registered schema.

**Original definition worth preserving:** S2 as universal topology is still right, provided we do not let folder ancestry become arbitrary Neo4j labels. Path ancestry is evidence; S2 is the validator.

**Current gap:** complete graph promotion from vault/world types, schema registry closure, and stronger graph-query/retrieval parity across CLI, gateway, and app surfaces.

### S3 / S3′ — Gateway, Sessions, And Temporal Runtime

**Current role:** [[S3]] owns the gateway/session/channel/runtime bridge. It is the source of truth for live session state.

**Prime law:** [[S3′]] owns [[Day]], [[NOW]], [[Kairos]], temporal context, presence, SpaceTimeDB/Graphiti runtime positioning, and bridge traffic across process/app surfaces.

**Original definition worth preserving:** the scientific loop language still belongs here as temporal operation grammar: observe, think, plan, build, execute, verify/learn. In current canon it should be framed as a runtime/agent workflow crossing S3/S4/S5, not as S3's sole identity.

**Current gap:** keep gateway protocol, session store, temporal projection, and Graphiti runtime law visibly separated from S5 meaning/review governance.

### S4 / S4′ — PI Runtime, Anima, VAK, And Capabilities

**Current role:** [[S4]] inhabits the system as PI agents: providers, auth, skills, teams, tools, permissions, and managed agent homes.

**Prime law:** [[S4′]] is [[Anima]]/[[VAK]] law: CPF/CT/CP/CF/CFP/CS evaluation, constitutional routing, [[Psyche]] continuity, team/subagent orchestration, and capability membranes.

**Original definition worth preserving:** "home of context" remains true if specified as agentic lived context rather than generic session state. Session transport is S3; inhabited context and dispatch are S4.

**Current gap:** role-scoped agent surfaces, capability governance, and VAK routing must remain distinct from S5 Epii review/autoresearch authority.

### S5 / S5′ — Epii, Review, Autoresearch, And World Return

**Current role:** [[S5]] returns system output to the world and back to itself through Gnosis, Nara, Epii, review, autoresearch, pedagogy, and promotion.

**Prime law:** [[S5′]] governs MEF/QL, review inbox, improvement candidates, kbase/gnosis interpretation, human-required promotion, and the Epii PI-agent embodiment.

**Original definition worth preserving:** S5 as synthesis and Möbius return is exactly right. What changed is the implementation surface: not "Notion/cloud" as authority, but Epii/world-return governance over real local and app-facing substrates.

**Current gap:** live Epii <-> Anima request execution, richer promotion semantics, and non-dry-run mutation through compiler/Hen law.

## Cross-Coordinate Relationships

### S1 / S2 / WorldTypes

[[S1′]] / [[Hen]] reads vault paths, frontmatter, wikilinks, MOCs, canvases, and source hashes. [[S2′]] validates what becomes graph state. [[World/Types]] folder hierarchy is therefore graph-promotion evidence, not arbitrary label authority.

### CT And T/T′

[[CT]] templates live under `World/Types/Coordinates/C/C1/C1'/CT/**` and flat [[World]] CT forms. [[T]] / [[T′]] thought artifacts live under `World/Types/Coordinates/T/**` and `Idea/Pratibimba/Self/Thought/**`.

Do not create standalone `World/Types/Templates` or `World/Types/Artifacts` authorities while these coordinate-native homes exist.

### Anima And Epii

[[Anima]] is the dispatch spine in [[S4′]]. [[Epii]] is the return spine in [[S5′]]. They may hand work to one another, but they do not share the same tool/skill authority surface.

| Spine | Coordinate | Function | Boundary |
| --- | --- | --- | --- |
| [[Anima]] | [[S4]] / [[S4′]] | VAK evaluation, dispatch, teams, tools, bounded execution | May deposit to Epii review/inbox, but does not resolve Epii governance. |
| [[Epii]] | [[S5]] / [[S5′]] | Review, autoresearch, pedagogy, Gnosis/Nara interpretation, promotion | May request Anima execution, but does not bypass S4 permissions. |
