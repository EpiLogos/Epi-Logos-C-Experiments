# [[M4']] Domain Spec

[[M4']] is [[Nara]]: journal, DAY/NOW lived context, dream, oracle, highlight, and personal Pratibimba continuity. It receives the harmonic clock as lived condition without exposing protected identity.

## User-Facing Surface

- Journal/flow editor as primary lived stream.
- Daily Note, Dream Journal, Oracle, and Highlight surfaces over one protected local flow substrate.
- DAY/NOW header showing current safe profile context: tick, mode/CF, coordinate, and deposition status where available.
- Highlight sendoff produces typed invocation envelopes for governed Anima/Epii paths.
- Playback of journal/oracle/cast trajectory can replay safe profile handles without exposing private content to public graph surfaces.

## Backend Contract Consumed

- [[S3]] / Khora owns DAY/NOW/session path and write authority.
- [[Graphiti]] stores episodic Nara memory under protected-local namespace rules.
- [[S0]] / [[S3]] profile projection provides safe public-current harmonic context.
- [[S2]] pointer anchor links public coordinate law without absorbing journal bodies.
- [[S4]] / [[S5]] agent invocation paths handle governed sendoff and review.

## Required `MathemeHarmonicProfile` Fields

- `tick`, `harmonic`, and `diatonic` fields for lived temporal/musical context.
- `pointerAnchor.coordinate` and relation provenance when a journal event is tied to a coordinate.
- `depositionAnchor.dayId`, `nowPath`, `sessionKey`, `episodeHandle`, and privacy class.
- Optional `mahamaya` fields for oracle/cast playback when M3 codec is available.
- Optional `planetaryChakral` fields for Nara modality display when S2 graph law supplies them.

## Privacy Boundary

M4' is the strictest boundary. Protected identity, private journal body, dream text, oracle interpretations, and unreconciled Graphiti episode bodies remain protected-local. Public profile data can be attached to entries as context handles. Promotion from private episode to canonical graph fact requires explicit Epii/S5' review.

## Visual / Audio Interaction Model

- Visual: journal and modality views show unobtrusive profile context, not a dashboard takeover. The clock can be replayed as trajectory metadata alongside entries.
- Audio: playback may sonify the safe profile sequence for a journal arc; it must not encode private text into audio analysis without governed local consent.
- Interaction: highlight/sendoff sends selected text, category, source range, DAY/NOW, coordinate, and profile handles through typed invocation envelopes.

## Readiness / Test Criteria

- Tests prove journal content is not present in S2 canonical graph projections or public profile payloads.
- Tests prove sendoff envelopes include DAY/NOW/session/profile handles and preserve selected text inside the protected invocation payload.
- Tests prove Daily Note, Dream Journal, Oracle, and Highlights persist/retrieve real modality data.
- Tests prove Nara distinguishes raw service connectivity from actual bounded agent access.
- Readiness is blocked until S3 deposition shape includes profile observation anchors and Graphiti privacy class.
