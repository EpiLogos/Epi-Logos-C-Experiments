## 0. Orientation

This file specifies the `M4-0-0` layer of the Nara Identity Matrix: the provisional **Birthdate Encoding** layer. Despite the working name, this layer does not only encode birthdates. It encodes the whole name/date identity packet: birth name, mythic or received names, birth dates, reception dates, word-components, letter streams, vowels, consonants, date components, and synthesis values. Its role is to transform name/date data into QL/MEF-readable identity light.

The output of this layer is **not** the final Nara identity. It is one contribution to the sixfold `M4-0` branch:

```text
M4-0-0  Birthdate Encoding / Pythagorean Derivations
```

The `M4-0-5` layer is where all prior layers are integrated at the elemental quaternionic level. `M4-0-0` should therefore produce a rich, traceable, non-flattened **elemental contribution**, not a final typology.

The shape is:


The central correction encoded here is this:

> MEF does not decorate a number after it has already been interpreted. MEF is the generative reading system. The numeric datum becomes readable only by being refracted through the 12 lenses and their internal sixfold positions. The lens content determines what kind of analysis is being performed.

## 2. Non-goals

This layer must avoid several tempting simplifications.

1. **Do not reduce a person to a single Pythagorean root.** The root is one signal among raw totals, compounds, mod6 positions, mod12 anchors, lens-square dynamics, and elemental projections.

2. **Do not treat MEF lenses as arbitrary interpretation tags.** The MEF lenses are the actual generative analysis types. L1 does causal analysis; L4 does phenomenological analysis; L1' does Jungian/phenomenal analysis; L2' does elemental/alchemical extraction; L5 and L5' read speech/Logos; L0' reads archetypal number.

3. **Do not collapse complexity at the end into a low-detail summary.** The final quaternion is an integration artifact, not a replacement for the evidence paths. Every quaternionic contribution should retain the chain by which it was produced.

4. **Do not make music the first derivation.** Music is a downstream rendering of the already-read MEF field. The correct order is reading → elemental quaternion → musical handoff.

5. **Do not treat L2' as a late add-on.** L2' is the element-bearing lens inside Square C, and Square C is a generative system of becoming: L2 Logical, L3 Processual, L3' Chronological, L2' Alchemical-Elemental. The elemental output is produced through the MEF system, not appended after it.

6. **Do not lock QL Music choices prematurely.** This layer should provide scale candidates, lens candidates, position vectors, motif material, and elemental timbral data. Final scale/mode/tuning/compositional choices belong to the QL Music layer and higher identity-song orchestration.

## 4. Pythagorean numeric extraction

### 4.1 Alphabet mapping

Use the standard modern Pythagorean cyclic alphabet:

| Number | Letters |
|---:|---|
| 1 | A, J, S |
| 2 | B, K, T |
| 3 | C, L, U |
| 4 | D, M, V |
| 5 | E, N, W |
| 6 | F, O, X |
| 7 | G, P, Y |
| 8 | H, Q, Z |
| 9 | I, R |

Normalization:

- Convert letters to uppercase.
- Ignore spaces and punctuation for total calculations.
- Treat hyphens as word-boundary markers, not numeric content.
- Preserve word boundaries for component analysis.
- Preserve received-name/birth-name distinction.
- Unicode transliteration should be normalized to ASCII unless the implementation has a dedicated Sanskrit/phonetic extension.

### 4.2 Y policy

`Y` is ambiguous. This layer should not hide the ambiguity.

Default policy:

```text
Y = consonant
```

Configurable policy:

```ts
type YPolicy = "consonant" | "vowel" | "phonetic" | "both";
```

Recommended behavior:

- Run the canonical reading with `Y = consonant` unless the name’s phonetics clearly require otherwise.
- If `YPolicy = both`, produce alternate vowel/consonant calculations and mark divergence points.
- For received or Sanskritic names such as `Satya` and `Jyoti`, default `Y` to consonantal /y/ unless overridden.
- For English names where `Y` participates in a vowel cluster, preserve an alternate reading if the user wants phonetic nuance.

## 5. QL positional mapping

For every numeric datum `n`:

```text
p6 = n mod 6
pInv = 5 - p6
axis = {p6, pInv}
c12 = n mod 12
```

### 5.1 Mod6 position and inverse

The mod6 value gives the internal QL position. The inverse must always be read alongside it:

| Direct | Inverse | Axis | Archetypal movement |
|---:|---:|---|---|
| P0 | P5 | 0/5 | source ↔ synthesis |
| P1 | P4 | 1/4 | emergence/definition ↔ context/integration |
| P2 | P3 | 2/3 | operation/process ↔ pattern/form |
| P3 | P2 | 2/3 | pattern/form ↔ operation/process |
| P4 | P1 | 1/4 | context/integration ↔ emergence/definition |
| P5 | P0 | 0/5 | synthesis ↔ source |

This produces a vector:

```text
V(n) = Pp6 → PpInv
```

Examples:

```text
n = 38: 38 mod 6 = 2 → P2→P3
n = 52: 52 mod 6 = 4 → P4→P1
n = 89: 89 mod 6 = 5 → P5→P0
```

The inverse is not a secondary afterthought. It is the counter-position that lets the datum become a living vector rather than a static label.

### 5.2 Mod12 chromatic/lens anchor

The mod12 residue gives the chromatic and MEF lens anchoring. The base chromatic order is semitone order from C:

| c12 | Pitch | MEF lens | Helix anchor |
|---:|---|---|---|
| 0 | C | L0 Quaternal | P0 / bimba |

Important distinction:


## 8. Full refraction model

The direct mod12 lens is important, but it is not the entire reading. The key rule is:

> Every datum’s mod6 position is passed through all 12 MEF lenses. The mod12 value then provides chromatic/lens anchoring and additional weighting.

This avoids the bug where only 12 of the 72 possible cells ever receive activation.

### 8.1 Matrices

Maintain at least four matrices:

```ts
type LensId =
  | "L0" | "L1" | "L2" | "L3" | "L4" | "L5"
  | "L0p" | "L1p" | "L2p" | "L3p" | "L4p" | "L5p";

type Position = 0 | 1 | 2 | 3 | 4 | 5;

type ResonanceMatrix = Record<LensId, Record<Position, number>>;
```

Matrices:

```text
R_full_pass       // every datum through every lens at p6 and inverse
R_anchor_bias     // mod12-selected lens and related lenses
R_square_diffuse  // Klein square propagation, Möbius return, X/X' partner, tritone mirror
R_total           // weighted sum of the above
```

### 8.2 Base activation

For each datum `d` with numeric value `n` and role weight `w`:

```text
p      = n mod 6
pInv   = 5 - p
lens   = lensByMod12[n mod 12]
square = squareOf(lens)
```

Full pass:

```text
for each MEF lens L:
    R_full_pass[L][p]    += w × fullPassFactor × roleLensAffinity(d.role, L)
    R_full_pass[L][pInv] += w × inverseFactor × fullPassFactor × roleLensAffinity(d.role, L)
```

Anchor bias:

```text
R_anchor_bias[lens][p]    += w × anchorFactor
R_anchor_bias[lens][pInv] += w × inverseFactor × anchorFactor
```

Square diffusion:

```text
for each relatedLens in squareOf(lens):
    R_square_diffuse[relatedLens][p]    += w × squareFactor
    R_square_diffuse[relatedLens][pInv] += w × inverseFactor × squareFactor

R_square_diffuse[mobiusReturn(lens)][p]       += w × mobiusFactor
R_square_diffuse[xPrimePartner(lens)][p]      += w × spandaFactor
R_square_diffuse[tritoneMirror(lens)][p]      += w × tritoneFactor
```

Recommended initial coefficients:

```text

inverseFactor  = 0.50
squareFactor   = 0.25
mobiusFactor   = 0.40
spandaFactor   = 0.35
tritoneFactor  = 0.30
```

These coefficients are provisional. The system should store them in configuration and expose them for calibration.

## 16. Implementation algorithm

### 16.1 High-level sequence

```text
1. Normalize identity packet.
2. Extract Pythagorean numeric data.
3. Build compound/root metadata.
4. For each datum, compute mod6 position, inverse, axis, mod12 lens anchor.
5. Create evidence path shell.
6. Pass each datum through full 12-lens MEF refraction.
7. Apply anchor, square, Möbius, X/X', and tritone propagation.
8. Aggregate R_total.
9. Generate square readings for A, B, C.
10. Project through L2' into elemental scores.
11. Build q_birthdate_encoding and Aether/Mineral caps.
12. Produce QL Music handoff candidates.
13. Return structured output with all evidence paths intact.
```

### 16.2 Pseudocode

```ts
function runBirthdateEncoding(packet: IdentityPacket, config: BirthdateEncodingConfig): BirthdateEncodingOutput {
  const normalized = normalizePacket(packet, config);
  const datums = extractPythagoreanDatums(normalized, config);

  const matrices = initMatrices();
  const evidencePaths: EvidencePath[] = [];

  for (const datum of datums) {
    const p = mod(datum.rawValue, 6) as Position;
    const pInv = (5 - p) as Position;
    const c = mod(datum.rawValue, 12);
    const anchor = lensByMod12[c];
    const w = datumWeight(datum, config);

    const path = initEvidencePath(datum, p, pInv, c, anchor);

    // Full MEF pass: all lenses read the datum's position.
    for (const L of ALL_LENSES) {
      const affinity = roleLensAffinity(datum.role, L, config);
      add(matrices.fullPass, L, p,    w * config.fullPassFactor * affinity);
      add(matrices.fullPass, L, pInv, w * config.fullPassFactor * config.inverseFactor * affinity);
      path.fullPassCells.push(cellId(L, p));
      path.fullPassCells.push(cellId(L, pInv));
    }

    // Anchor bias from mod12.
    add(matrices.anchorBias, anchor.lens, p,    w * config.anchorFactor);
    add(matrices.anchorBias, anchor.lens, pInv, w * config.anchorFactor * config.inverseFactor);

    // Lens-square diffusion.
    for (const L2 of squareOf(anchor.lens).lenses) {
      add(matrices.squareDiffuse, L2, p,    w * config.squareFactor);
      add(matrices.squareDiffuse, L2, pInv, w * config.squareFactor * config.inverseFactor);
    }

    add(matrices.squareDiffuse, mobiusReturn(anchor.lens), p, w * config.mobiusFactor);
    add(matrices.squareDiffuse, xPrimePartner(anchor.lens), p, w * config.spandaFactor);
    add(matrices.squareDiffuse, tritoneMirror(anchor.lens), p, w * config.tritoneFactor);

    evidencePaths.push(finalizePath(path, matrices));
  }

  const total = combineMatrices(matrices);
  const mefReading = analyzeMef(total);
  const elemental = extractElementalContribution(total, evidencePaths, config);
  const musicHandoff = deriveMusicHandoff(datums, total, elemental, config);

  return {
    layerId: "M4-0-0",
    layerName: "Birthdate Encoding",
    input: packet,
    config,
    pythagorean: buildPythagoreanReport(datums),
    ql: buildQlReport(datums),
    mef: { matrices: { ...matrices, total }, ...mefReading },
    elemental,
    musicHandoff,
    evidencePaths,
    warnings: collectWarnings(datums, config)
  };
}
```
