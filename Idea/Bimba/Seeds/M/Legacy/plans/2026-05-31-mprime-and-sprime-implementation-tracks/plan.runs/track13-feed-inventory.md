## §5 — Recommended Track 13 Sequencing

The cleanup ordering matters because most adapter callers depend on the shims, the shims hide which laws are duplicated, and duplicated-law unification requires the migration paths to be clear first.

1. **First — inventory recast (T0 in plan)**: Land this file's content as `Body/S/S0/epi-cli/contract-inventory/` machine-readable record (Track 13 T0 deliverable). This is the substrate-truth artifact every later tranche reads.
2. **Second — parity vocabulary fix (T1)**: Recast `gate/parity.rs::CoordinateParityStatus::Mirror` into explicit `Adapter` / `CompatibilityAdapter` / `TemporaryLiveHost`. Cheap, high leverage — every later tranche references the new vocabulary.
3. **Third — shim removal first, BEFORE adapter cleanup**: Shims (§2) often keep adapter callers happy. Removing shims first reveals which adapters are genuinely load-bearing vs which only exist because a shim was masking them. Order: remove "documented-as-deprecated" shims (B, H, I) first; then CLI-visible legacy aliases (C, D, E) with one beta-cycle warning; then stubs as their real implementations land (J, K, L); leave A, F, G for last (cross-track dependencies).
4. **Fourth — duplicated-law unification of the high-divergence-risk surfaces (4-i, 4-iii, 4-vii)**: These fail silently — fix before adapter cleanup so that any extracted code lands in the unified law site, not a third duplicate location.
5. **Fifth — adapter extraction (T2 → T7)**: Move non-trivial adapter behavior out (server.rs route table → S3, sessions JSON envelope → S3, mediation-route law → S4, improve cross-store guard → S5, etc.). At this point every adapter has a known successor.
6. **Sixth — temporary-live-host migration (T3, T4 territory)**: Decide the gateway process question (α) and the Graphiti port-37778 question (β) — these are architectural choices, not refactors.
7. **Seventh — CI guardrails (T9) and final report (T10)**: Once the membrane is clean, add tests that fail when S0 grows new downstream service-law modules without an adapter annotation.

Throughout: keep behaviour-preserving discipline (existing CLI commands and gateway methods must not change wire shape during cleanup) — the inventory recast and parity vocabulary fix do not require any behavior change.
