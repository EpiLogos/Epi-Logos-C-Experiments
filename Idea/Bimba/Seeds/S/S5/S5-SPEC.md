# S5/S5' Specification: Integral World Boundary and Epii Return Law

## Status

This is the consolidated S5-level master specification. It replaces the older scattered [[S5]], [[S5']], [[S5'Cx]], and S5-y/S5-y' files as the build reference for the world-boundary and return layer.

[[S5]] is the integral world-boundary: the layer where the system addresses the world, ingests the world's return, navigates the [[Bimba]] map as a living oracle, runs [[Gnosis]] / [[RAG-Anything]] corpus flows, invokes [[Graphiti]] episodic memory through the S3' temporal runtime, surfaces [[NotebookLM]] / [[Vimarsa]] / [[kbase]], and exposes [[M']] functions through [[Nara]] and related engines.

[[S5']] is [[Epii]]: the separate PI agent instance and governance law for deep knowledge, [[MEF]] lenses, [[QL]] evaluation, gnosis strategy, episodic usage, pedagogy, autoresearch, improvement, and the [[Möbius Return]]. It is also the user-position in the system: the place where [[Nara]] / [[M4]] [[Pratibimba]] and [[PASU]] evolution fold into [[Epii]] / [[M5]] reflective self-research. [[Aletheia]] remains in [[S4.5']] as the UX membrane and crystallisation trigger inside [[Anima]]'s runtime; [[Epii]] is the deep backend, user-representative oracle, and the one to which [[Anima]] / [[Aletheia]] work becomes intelligible.

[[Epii]] is not merely the final tool called by [[Anima]], and [[Anima]] is not merely an executor subordinate to [[Epii]]. They are distinct PI-agent embodiments of the same spine grammar. [[Anima]] carries the dispatch spine through [[Pleroma]] and ta-onta; [[Epii]] carries the return spine through the [[epi-logos plugin]] and [[autoresearch]]. Epii may invoke Anima for implementation, dispatch, validation, or runtime action; Anima may contact Epii for inbox deposits, meaning review, consent/validation, and improvement requests.

The core planned spine for S5/S5' is the [[epi-logos plugin]]. That plugin is being developed outside this immediate spec pass, but it is the intended resource and skill body for executing [[Epi-Logos]] philosophy and the related [[Nara]] / [[Epii]] functionalities. It should hold the core philosophical resources, skills, prompt contracts, M' function affordances, and Epii/Nara-facing operational grammar. External automations, [[Notion]], webhooks, and publication connectors remain valid S5 manifestations, but they are secondary to this plugin-first planned state.

The dynamic self-improvement spine of the [[Epii]] agent is the [[autoresearch]] loop. Because Epii is the #5 [[epi-logos]] return position, autoresearch is how the prior architecture turns back upon itself: hypotheses, challengers, evaluations, keep/discard decisions, promotions, and schema/skill refinement return through S5' into lower layers. This is not a generic background loop; it is Epii's reflective capacity, informed by [[Sophia]] disclosures and [[Anima]] / [[Aletheia]] outputs.

This is the newest and least-settled S-level. The older [[Notion]] / dashboard files preserve useful #5 intuition, but the current architecture is broader: [[Notion]] is one possible S5 manifestation, not the coordinate itself.

### Corrections / re-homing

The current architecture supersedes several older assumptions:

- [[S5]] is not [[Notion]]. Notion remains an optional external/pedagogical manifestation.
- [[S5]] is not raw sync alone. It is the full world-boundary: outward publication plus inward world-return.
- [[Aletheia]] is not the whole S5'. It remains [[S4.5']] as UX membrane, thought/T-bucket handling, and crystallisation trigger inside [[Anima]].
- [[Epii]] is [[S5']]: a separate PI instance, user-representative deep oracle, [[Bimba]] navigator, M' host, gnosis governor, pedagogue, and reflective improvement law.
- [[S5]] / [[S5']] enacts both [[M4]] and [[M5]]: [[Nara]] holds the personal/dialogical [[Pratibimba]] and [[PASU]] evolution field, while [[Epii]] holds the developer/reflection portal that aligns the system with its own philosophy.
- The [[epi-logos plugin]] is the planned S5/S5' spine for core resources and skills. It is the execution body for Epi-Logos philosophy, Nara functions, Epii oracle work, and related pedagogical/disclosure affordances.
- [[autoresearch]] is the main dynamic spine of the Epii agent. Its whole-system reach follows from Epii's #5 return position, not from being a free-floating infrastructure service.
- Human review gates, validation needs, and user-consent thresholds should route to an Epii-accessible review inbox. That inbox is the point at which [[Anima]] dispatch results and [[Aletheia]] crystallisations are made meaningful for the user-position, not merely a task queue.
- [[Graphiti]] is architecturally [[S3']] temporal episodic runtime/library. [[S5]] / [[S5']] owns invocation, usage, search strategy, arc governance, disclosure, and reflective meaning. The current HTTP wrapper is not target architecture.
- [[Gnosis]] / [[RAG-Anything]] uses [[S2]] graph/vector substrate but belongs here as world-return corpus service.
- [[M']] functions are surfaced through `epi nara` today, but their integrative knowledge home is S5/S5' through [[Epii]].
- [[bimba-mcp]] remains an S2' external graph interface, not the PI agent's internal S5 oracle.

## D. Key Architectural Decisions

1. [[S5]] is the integral world-boundary, not [[Notion]] and not raw sync.

2. [[S5']] is [[Epii]], a separate PI instance and the user's integrative representative in the system.

3. [[S5]] / [[S5']] enacts both [[M4]] and [[M5]]: [[Nara]] holds the personal [[Pratibimba]] / [[PASU]] evolution field, while [[Epii]] holds the reflective developer portal that performs [[epi-logos]] over the system.

4. The [[epi-logos plugin]] is the planned S5/S5' spine. A local scaffold now exists at `Body/S/S5/plugins/epi-logos`; it should grow into the complete core resource and skill body for executing Epi-Logos philosophy and the Nara/Epii functional field.

5. [[autoresearch]] is the dynamic self-improvement spine of the [[Epii]] agent. Its effects return through every S-level because Epii is the #5 reflective/user position, not because autoresearch is a generic ambient service.

6. The Epii review inbox is the canonical recipient for [[Anima]] results, [[Aletheia]] crystallisations, [[Sophia]] disclosures, and human validation gates. It is where their work makes sense to the user-position and where review resolves into Nara action, autoresearch, SEED carry-forward, or promotion.

7. [[Aletheia]] remains [[S4.5']] as UX membrane and crystallisation trigger; deep knowledge governance belongs to Epii/S5'.

8. [[Graphiti]] architecture belongs to [[S3']] as temporal episodic runtime/library. S5/S5' owns Graphiti invocation, search, arc governance, reporting density, and reflective meaning.

9. [[Gnosis]] / [[RAG-Anything]] belongs to S5 as world-return corpus service using S2 substrate.

10. [[kbase]] / [[Vimarsa]] belongs to S5' as aperture and context-pool governance, even if current command plumbing is script-backed.

11. [[M']] functions should be typed S5 methods over the existing [[Nara]] operator surface, not swallowed by S4 agent runtime.

12. [[Notion]], external automations, webhooks, and publication surfaces remain alive as secondary S5 connectors, not the present spine.

13. The S5 build must be production-first: real ingestion, real retrieval, real embeddings, real graph/vector stores, real review states, and real improvement evaluations. Mock-only tests would be misleading at this layer.

14. Completion of S5/S5' returns the whole S-level pass to [[S0]] / [[S0']]: command shape, API routing, bootstrap, test harness, and audit trail must be made executable.
