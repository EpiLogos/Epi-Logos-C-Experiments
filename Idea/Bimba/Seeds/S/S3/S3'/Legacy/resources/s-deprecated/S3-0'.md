# S3.0': Khora API (Environment & Hooks)

> **QL Position**: #0 - Ground/Relational Field

## Description

Hook system, environment initialization, runtime state - Khora provides the ground where all plugins come into being. Khora is the **receptacle** that receives all plugin manifestations and provides the space where forms come into being.

### Memory Type Descriptions

**EPISODIC (P0 Ground)**: Personal experiences, specific events, temporal sequences
- Storage: Daily notes, session transcripts (JSONL), CONTEXT.md files
- Refresh: Daily, per-session
- Access: S1' monadological operations

**SEMANTIC (P5 Integration)**: General knowledge, concepts, patterns, archetypes
- Storage: Bimba Seeds, World models, Neo4j graph
- Refresh: Continuous via GraphSync
- Access: S2' graph operations with coordinate filters

**WORKING (P4 Context)**: Current session state, active task context
- Storage: CONTEXT.md files, bootstrap file injection
- Refresh: Per-session
- Access: Khora `load_context()` → agent bootstrap

**PROCEDURAL (P2 Operation)**: Skills, workflows, operational patterns
- Storage: Skills library (SKILL.md), skill snapshots
- Refresh: At agent turn start (frozen snapshot)
- Access: S3-2' Pleroma skill invocation

**DECLARATIVE (P1 Definition)**: Definitions, specifications, metadata
- Storage: Form files, Type definitions, seed files
- Refresh: On seed update
- Access: S1' `parse_form()` extraction

**REFLECTIVE (P5 Integration)**: Learning extracts, crystallized wisdom
- Storage: Daily note #5 sections, `/Thought/Insight/` folder
- Refresh: Daily synthesis (Möbius return)
- Access: S3-5' Aletheia learning extraction

## Core Operations

```python
class KhoraAPI:
    # Plugin lifecycle
    def init_plugin(plugin_name: str) -> PluginResult
    def verify_environment() -> EnvironmentStatus

    # Hook system
    def register_hook(hook_name: str, handler: Callable) -> HookID
    def trigger_hook(hook_name: str, context: dict) -> HookResult

    # Context persistence (Working Memory)
    def persist_context(file_path: str, content: str) -> bool
    def load_context(file_path: str) -> Optional[str]

    # Bootstrap injection
    def inject_bootstrap(session_id: str, files: List[BootstrapFile]) -> bool
    def get_bootstrap_context(session_id: str) -> BootstrapContext
```

## Hooks Provided

| Hook | Event | Purpose | Memory Type Affected |
|------|-------|---------|---------------------|
| SessionStart | Session init | Initialize session context | Working |
| SessionStop | Session end | Extract learnings, archive | Reflective |
| PreToolUse | Before tool | Tool validation | Procedural |
| PostToolUse | After tool | Result capture | Working |
| UserPromptSubmit | User input | Input processing | Episodic |
| PreResponse | Before AI | Response validation | Working |
| PostResponse | After AI | Response logging | Episodic |

## Bootstrap Files (Working Memory Injection)

Khora injects context at session start via `agent:bootstrap` hook:

| File | Purpose | Source |
|------|---------|--------|
| `CONTEXT.md` | Runtime session state | Daily note + active entry |
| `QL-PARADIGM.md` | Coordinate system instruction | Bimba seeds |
| `SOUL.md` | Agent persona (optional) | Agent config |
