## Build Contract

Pleroma must expose executable routes without seizing ontology. It can provide `epi_cli`, `tmux`, `cmux`, `worktrunk`, and other bounded primitives; [[Hen]] owns vault law, [[Anima]] decides dispatch, [[Khora]] writes, and [[S0]] executes process commands. The permission boundary must be inspectable before a tool runs.

The [[oh-my-codex]] lineage belongs here as a future/native Pleroma substrate, but current docs must not pretend the fork is complete. The live package is a working adapter/package surface; the ta-onta S4.2' carrier remains the canonical module.
