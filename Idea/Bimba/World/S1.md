## #0 Ground

The foundation of the material container: the filesystem itself. Obsidian is built on markdown files stored in a vault folder. The `.obsidian/` configuration, the file hierarchy, the actual bytes on disk — this is the ground from which all semantic meaning emerges.

## #1 Definition

**S1 is the Material Container** — where flat-file markdown storage meets YAML frontmatter property systems, enabling human-readable, version-controllable knowledge with wiki-link based relationships. It is both the repository of all content AND the relationship network that gives content meaning.

S1 operates in two modes:
- **Bimba**: Canonical specifications (templates, seeds, World forms)
- **Pratibimba**: Instance/reflection (actual notes, tasks, daily entries)

## #3 Pattern

The **Bimba-Pratibimba Pattern**: Canonical specifications (Bimba) in flat `/World` folder → instance reflections (Pratibimba) in nested `/Pratibimba` structure.

The **Template Lifecycle**: Seed → Bimba Kanban → Pratibimba Template → Instance Files

The **Möbius Return**: Instance feedback (synthesis) → Template evolution → Seed refinement → New generation instances improved

The **Non-Dual Data Unit**: Content (perception) + Wikilinks (appetite) + Context (folder/situation) = Complete monad

## #4 Context

**Temporal**: Obsidian developed 2020+, following digital garden / PKM movement. Fits modern knowledge work within human timescale.

**Spatial**: Obsidian vaults can exist anywhere — local filesystem, synced clouds, git repositories. The vault is the primary unit of knowledge.

**Cultural**: The Obsidian community emphasizes human-readable, version-controllable, portable knowledge. No vendor lock-in. Privacy-first. Open plugins ecosystem. Connected notes paradigm.

## Key Distinctions

- **Canvas (C3 Process)** = Entity AS process, frozen in workspace form - work here without committing to synthesis
- **Template fields** IN synthesis file define the processual dimension/directionality
- **Pratibimba** = Composite of positions per task context, represented in/as synthesis file, but actual form lives in `/thoughts/` and task directories
- **Date-time** = Anchor for pratibimba, **Links** = Ligature to Bimba files
- **CONTEXT.md** = Canonical type or part of seed (runtime context), NOT pratibimba template (operates outside Bimba structure)
- **Pratibimba templates** = ONLY for Claude when composing workflows/commands
- **Ad hoc files** = Genuinely ad hoc files (task-specific, ephemeral)
