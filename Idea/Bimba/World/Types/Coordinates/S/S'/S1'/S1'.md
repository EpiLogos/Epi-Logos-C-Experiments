# [[S1']]: Content API / Monadological Mapping

> The implicate contract surface of the vault: each file as monad, each template as frozen process, each invocation as coordinate-bearing artifact.

## #5 Integration — What This Layer IS

[[S1']] is the formal content law of the stack. If [[S1]] is the material container, [[S1']] is the implicate grammar by which files become intelligible as coordinates, templates, contexts, processes, and returns. It is the content API not in the narrow CRUD sense but in the monadological sense: each file perceives, links, and is situated; each artifact is a local crystallisation of the whole.

Its normative operation set is the sixfold monadological API:

- `init_ground()` — establish the container, folder, or context horizon
- `parse_form()` — extract what a file IS as form
- `read_content()` — perceive content, appetite, and relation through `[[wikilinks]]`
- `structure_pattern()` — freeze process in canvas and template structure
- `coordinate_context()` — situate the artifact in type, family, temporal, and nested contexts
- `integrate_links()` — return the artifact into backlinks, synthesis, and larger harmony

This layer also embeds the C-level manifestations that older docs scattered into [[S1'Cx]]:

- [[C0]] at [[S1']] is folder-ground and canonical domain residency
- [[C1]] is the flat form in [[World]]
- [[C2]] is the frontmatter-bearing entity surface
- [[C3]] is the canvas as frozen process
- [[C4]] is the type folder plus MOC index
- [[C5]] is the invoked and reflective artifact in [[Present]] and [[Pratibimba]]

The template system belongs here as law. [[CT]] names content causes; [[CT']] names frozen processes. [[Daily Note]] is the metatemplate of homebase context; [[NOW]] is its bounded child form; [[Thought]] is the [[CT5]] / Möbius return surface; [[CT4b']] is the master temporal-fractal template family for shared day-context operations. In the eventual shape, runtime files are not ad hoc notes but invocations of canonical template families with explicit lineage, profile, and coordinate law.

The homebase law is equally central. [[S1']] should make one top context available for living work: the day-context as shared horizon, with nested [[NOW]] invocations, task lanes, pattern surfaces, contextual embeddings, and synthetic return. The user should not juggle scattered contexts; the template system should hold the whole.

## Cross-Layer Actualisation

[[Hen]] is the primary actualiser of [[S1']]: template registry, frontmatter law, content typing, breadcrumb generation, and vault-to-graph coordination all belong to its remit. [[Khora]] provides the write primitive and persistence edge. [[Chronos]] consumes [[S1']] as the temporal lifecycle contract of [[Day]], [[NOW]], and archival return. [[Anima]] consumes it as the content membrane for agentic runtime. [[Aletheia]] and [[S2']] consume it as the layer where links, provenance, and form are already explicit enough to be checked, traversed, and crystallised.

## #0 Ground — What Returns to Source

[[S1']] returns to source as a set of binding requirements:

- every frontmatter field must obey coordinate law, with `coordinate` as the one bare ground key
- every template invocation must preserve lineage from canonical family to concrete artifact
- every temporal artifact must be a real context-form, not a random filename with accidental semantics
- every relation in content should be legible both as `[[wikilink]]` and as future graph structure
- every named artifact family should be traceable to its [[CT]] / [[CT']] ground

Sub-coordinates should eventually make explicit the full API contract for each monad operation, the complete homebase architecture of [[Day]] and [[NOW]], the full template matrix from [[CT0]] through [[CT5]], and the precise vault-to-graph parity law. Any runtime that falls short of this is unfinished implementation, not specification truth.
