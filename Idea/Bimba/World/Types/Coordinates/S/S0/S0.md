# [[S0]]: Terminal / CLI / C Ground

> The ground execution layer: the common substrate on which every later [[S]]-layer must become runnable, testable, and distributable.

## #5 Integration — What This Layer IS

[[S0]] SHALL be the lowest explicit layer of the [[S]] stack. It names not a single shell window or one binary, but the entire terminal-facing execution plane through which [[Epi-Logos]] becomes materially operable.

The eventual shape of [[S0]] is this:

1. [[Filesystem]] ground. All vault roots, repo roots, cache roots, generated artifacts, and runtime state must be concretely placeable here.
2. [[Process]] ground. Every operation in higher layers must be reducible to explicit spawn, environment, stream, signal, and exit-code semantics.
3. [[Command]] ground. The system must possess a stable executable surface through which humans and agents alike can invoke the stack.
4. [[Build]] ground. Compilation, verification, packaging, installation, and bootstrap all belong to [[S0]].
5. [[C]] ground. The compiled substrate, including the coordinate kernel and its memory laws, belongs here as the irreducible bedrock beneath all reflective routing.

[[S0]] therefore has a strict telos: it is the layer of executable honesty. Whatever is promised in [[S1']], [[S2']], [[S3']], [[S4']], or [[S5']] must cash out here as actual runnable surface, actual persistence, actual I/O contract, and actual reproducible build path. [[S0']] may name and route this ground, but [[S0]] is what makes the naming true.

Its inner sixfold should crystallise as:

- [[S0-0]]: [[Filesystem]] and environment ground
- [[S0-1]]: command atoms and executable discovery
- [[S0-2]]: streams, pipes, serialization, and transformation
- [[S0-3]]: scripts, wrappers, composition, and automation entrypoints
- [[S0-4]]: session runtime, locks, pidfiles, local state, and managed process context
- [[S0-5]]: packaging, installation, release, and bootstrap return

## Cross-Layer Actualisation

[[Khora]] ([[S4.0']]) actualises [[S0]] as bootstrap, environment materialisation, session identity, and canonical write authority. [[Hen]] ([[S4.1']]) actualises [[S0]] as the file and template substrate required for vault life. [[Pleroma]] ([[S4.2']]) actualises [[S0]] as skill projection and executable affordance. [[Chronos]] ([[S4.3']]) actualises [[S0]] as temporal invocation and scheduled run surface. [[Anima]] ([[S4.4']]) actualises [[S0]] as dispatch substrate for agent runtime and orchestration. [[Aletheia]] ([[S4.5']]) actualises [[S0]] as verification, gatekeeping, and reflective promotion.

All six [[ta-onta]] extensions therefore consume [[S0]]. None float above it.

## #0 Ground — What Returns to Source

What must return to source in [[S0]] is the law of the layer:

- [[S0]] MUST remain small, explicit, and composable.
- [[S0]] MUST NOT absorb semantic authority that belongs to [[S0']].
- [[S0]] MUST NOT absorb content authority that belongs to [[S1']].
- [[S0]] MUST remain harness-agnostic: shells and executables may vary, but the ground law remains stable.
- [[S0]] MUST be verifiable through real execution and real tests, because false grounding at [[S0]] contaminates the whole stack.

The return of [[S0]] is simple: all higher articulation collapses back to executable substrate. If a thing cannot be spawned, built, persisted, addressed, or verified here, it is not yet real in the stack.
