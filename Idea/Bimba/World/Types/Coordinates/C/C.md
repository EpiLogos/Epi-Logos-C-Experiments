# C Layer Typology

## Scope

[[C]] is the primary typology layer inside [[World/Types]]. Other coordinate families qualify and locate a thing; the C layer says what kind of thing the thing is.

This is where coordinate becomes language, category, and reusable type law.

## C0-C5 Semantic Spine

| Coordinate | Semantic type authority | Primary function |
| --- | --- | --- |
| [[C0]] | Source / Ground | Roots, namespaces, source conditions, coordinate ground. |
| [[C1]] | Forms / Templates | Forms, definitions, CT templates, markdown authoring forms. |
| [[C2]] | Entities / Properties / Tags | Entity candidates, aliases, properties, tags, and relation-field evidence. |
| [[C3]] | Processes / Canvases / Diagrams | MOC canvases, ideal canvas forms, architecture diagram forms, process diagrams. |
| [[C4]] | Types / Contexts / MOCs | Type authorities, folder MOCs, context frames, Dataview/query indexes, residency classes. |
| [[C5]] | Crystallisations / Pratibimba | Flat World graduation, integration forms, reflection/archive handoff. |

## Rule

Semantic categories belong under their owning C coordinate. Do not create top-level `World/Types/Templates`, `Entities`, `Properties`, `Diagrams`, or `Artifacts` roots while these C-native homes exist.

## Hen Workflow

1. [[Hen]] resolves an artifact's category through C first.
2. The owning C folder supplies semantic type context and MOC/canvas evidence.
3. Other coordinate families supply system, lens, position, thought, or subsystem qualification.
4. Stable type-local knowledge graduates to flat [[World]] only after review.

## C Prime Audit

The reflective C-prime grammar must remain aligned with [[CPF]], [[CT]], [[CP]], [[CF]], [[CFP]], and [[CS]]. Any apparent mismatch between folder ancestry and this order is a Cycle 3 audit item before Hen treats that branch as graph-authoritative.
