# Datasets Type MOC

## Scope

[[Datasets]] indexes graph/corpus/cache/source datasets as type evidence for [[S2]] graph law and [[S5]] world-return governance.

## What Belongs Here

- Bimba map datasets, Neo4j graph data, [[Graphiti]] episodes/arcs, [[Gnosis]] corpora, RAG sources, semantic cache records, contract ledgers, and migrated legacy corpora.

## What Does Not Belong Here

- Raw temporary exports without provenance.
- Personal or protected material lacking residency classification.

## Flat World Forms

Stable dataset concepts may crystallise into flat [[World]] definitions, but raw data remains in its owning dataset/source location with provenance links.

## Seed Evidence

Read [[S2-SPEC]], [[S5-SPEC]], [[S-SOURCE-TRACEABILITY-INDEX]], and [[LEGACY-DOCS-MIGRATION-INDEX]] before changing dataset typology.

## MOC/Canvas Rule

This file is the textual MOC. [[Datasets.canvas]] is the visual MOC. Dataset subfolders must declare source, privacy, coordinate owner, and graph-promotion status.

## Graph Label Intent

Dataset ancestry should become validated dataset/source evidence and relationships through [[S2]], not arbitrary folder labels.

## Open Gaps

- Dataset privacy/residency classes are not yet normalised across migrated material.
- Graph promotion for dataset families needs explicit Hen/S2 fixtures.
