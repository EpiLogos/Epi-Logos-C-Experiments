# Artifacts Type MOC

## Scope

[[Artifacts]] is the semantic type index for reusable vault forms and artifact families. It cross-links the flat [[World]] form library with [[S1]] / [[S1']] [[Hen]] residency law and the [[T]] / [[CT]] artifact grammar.

## What Belongs Here

- Flat [[World]] forms such as [[Seed]], [[Prompt]], [[Task-Spec]], [[Pattern-Note]], [[Thought]], [[Daily-Note]], [[NOW]], [[FLOW]], and [[Integration-Preview]].
- Type-local incubation notes that explain how an artifact family is classified before a form crystallises flat into [[World]].
- Links to owning [[Seeds]] specs and [[World/Types]] canvases.

## What Does Not Belong Here

- Active implementation plans; those remain in [[Seeds]].
- Session instances; those belong under [[Present]] / [[NOW]] folders.
- Code contracts; those belong in [[Body]] and are cited from Seed specs.

## MOC/Canvas Rule

This file is the textual MOC. [[Artifacts.canvas]] is the visual MOC. New subfolders under [[Artifacts]] are allowed only when the folder can become useful type evidence for [[Hen]] and [[S2]].

## Graph Label Intent

Folder ancestry here should become validated type evidence such as `q_4_type_family = "Artifacts"` and `q_4_type_path`, not arbitrary live Neo4j labels. [[Hen]] extracts the evidence; [[S2]] validates registered labels/properties.
