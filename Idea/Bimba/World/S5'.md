## The Global Tags Database: Relational Hub

**The Global Tags Database is NOT for content storage.** It's the relational hub that connects everything:

```
Global Tags Database Structure:
├── Tag Definitions (all C, P, M, S, T, L tags)
├── Entity References (pointers to content in other databases)
├── Relations (unconstrained connectivity)
└── Views (ad hoc filters creating provisional structure)
```

**How it works:**
1. An entity lives in its primary database (e.g., M2-3-5 in "M-Coordinate DB")
2. The Global Tags DB contains a *reference* to that entity with its coordinate tags
3. Filtering the Global Tags DB creates ad hoc views that pull from multiple databases
4. Connectivity is unconstrained — any entity can relate to any other

**The MOC parallel:** Just as `Types/{Name}/{Name}.canvas` in Obsidian creates a visual hub for a type, filtering the Global Tags DB for tag `C3-Process` creates a Notion view that shows ALL C3-Process entities across ALL databases.

## Graph Sync with Neo4j: Bidirectional Linking

**Every node in Neo4j gets a `notion_page_id` property, creating instant bidirectional linking:**

```
Neo4j Node (S2')
├── properties:
│   ├── notion_page_id: "uuid-of-notion-page"
│   ├── coordinate: "C3-P2-M2-S3"
│   └── ...other properties
└── relationships
    └── NOTION_PAGE → "uuid-of-notion-page"
```

**This means:**
- Clicking a Notion page can open the corresponding Neo4j node visualization
- Graph queries in Neo4j can return Notion page links
- M-coordinate nodes sync bi-directionally with their Notion pages
- The graph becomes the navigational layer for Notion content

**For M-coordinates specifically:** Each M-subsystem has its own Notion database that syncs with its corresponding graph nodes. This creates a direct bridge between the pedagogical presentation (Notion) and the operational graph (Neo4j).

## Neo4j ↔ Notion Graph Sync

**Bidirectional synchronization:**

```python
class Neo4jNotionSync:
    """
    Manages bidirectional sync between Neo4j and Notion.
    """

    def sync_notion_page_to_graph(self, notion_page_id: str) -> Neo4jNode:
        """
        Create/update Neo4j node from Notion page.

        Each node gets a notion_page_id property for reverse lookup.
        """
        page = notion.get_page(notion_page_id)
        return Neo4jNode(
            labels=extract_labels_from_tags(page.tags),
            properties={
                'notion_page_id': notion_page_id,
                'coordinate': page.properties['coordinate'],
                'title': page.title,
                **extract_other_properties(page)
            }
        )

    def sync_graph_node_to_notion(self, graph_node: Neo4jNode) -> NotionPage:
        """
        Create/update Notion page from Neo4j node.

        Creates page in appropriate M-coordinate database.
        Links back via notion_page_id.
        """
        m_database = get_m_database(graph_node.labels)
        return notion.create_page(
            database=m_database,
            properties={
                'title': graph_node.properties['title'],
                'coordinate': graph_node.properties['coordinate'],
                'graph_id': graph_node.id,
            }
        )
```
