# Track 09 - Agentic Mediation And Operational Capacities Implementation Plan

This track makes M5-4 the governed mediation bridge between the M5-2 S/S' substrate and the M5-3 Tauri/Theia surfaces. It does not grant agents ad hoc powers: Sophia, Anima, Pi, and Aletheia operate only through real S0/S0' profile and VAK contracts, S2 graph contracts, S3 gateway/session/event contracts, S4 PI runtime contracts, and S5/S5' review/autoresearch contracts, with `/body` receiving lightweight affordances and `/pratibimba/system` receiving the deep Agentic Control Room.

## Goal

Deliver a production-ready M5-4 mediation layer that can evaluate VAK coordinates, route work through Anima, execute bounded Pi/Sophia/Anima/Aletheia workflows, collect evidence from real S-stack services, deposit review/autoresearch records through S5, and surface operational capacity state in the Tauri/Theia IDE without bypassing human gates, privacy classes, namespace boundaries, or promotion law.

The first buildable outcome is not "agents can do anything." The first outcome is a tested mediation path: selected artifact and coordinate context -> real `vak_evaluate` result -> Anima route -> bounded agent run envelope -> S0/S2/S3 evidence attachment -> S5 review/autoresearch deposit -> IDE control-room inspection -> governed review or dry-run promotion. Each of the six operational capacities must have at least one real end-to-end readiness path into that flow: Anuttara, Paramasiva, Parashakti, Mahamaya, Nara, and Epii-on-Epii.

## Architectural Keystones

- **M5-4 is a bridge, not a sovereign backend.** M5-4 does not compute profile law, graph law, SpaceTimeDB truth, review state, or promotion authority. It mediates across S0/S2/S3/S4/S5 contracts and makes their evidence usable by agents and surfaces.
- **VAK is the execution gate.** Every non-trivial mediated run starts with CPF/CT/CP/CF/CFP/CS, maps CF through Anima, and records the route decision. CF `(0000)` invokes Nous-style clearing and returns for re-evaluation; CPF `(00/00)` blocks autonomous dispatch and asks for user-facing brainstorming. The implementation must use the real `epi agent vak evaluate` and S4 Anima route surfaces, not a UI-only dropdown. The `dispatch_tools[]` registry in `capability-matrix.json` codifies this discipline at the runtime layer: every dispatch tool (`dispatch_agent`, `dispatch_parallel_agents`, `dispatch_fusion_agents`, `dispatch_moirai_night_pass`, `anima_self_invoke`, `run_chain`) carries `upstream_required: ["vak-evaluate"]` — dispatch without a real prior VAK evaluation is rejected at the gate, not as advisory. CT (Content-Type) is recorded as CT4a or CT4b per the CT4 harmonization, so route decisions distinguish the two registers.
- **Agent roles are differentiated and testable.** Sophia is the default long-arc wisdom/governance lead; Anima is the dispatch grammar and Nara-primary voice governor; Pi is the bounded technical dispatcher and evidence preparer; Aletheia is the disclosure/crystallisation tracker and S4.5'/S5 handoff bridge. No agent may approve human-required gates, mutate raw runtime state, or treat another agent's authority as its own.
- **Aletheia is not a flat source label.** S5 may keep coarse `Aletheia` source compatibility, but every S4.5 disclosure that drives mediation must preserve differentiated Aletheia lineage: Anansi orientation, Moirai Klotho/Lachesis/Atropos Night' modes, Janus temporal envelope, Mercurius translation, Agora aggregation, Zeithoven manifestation, tool refs, skill refs, graph namespace refs, DAY/NOW/session refs, gate refs, privacy class, and readiness. Flattening this to a generic review source is an acceptance failure.
- **The six operational capacities are first-class route profiles.** Anuttara is construction-not-training with Sophia lead and user-final gates for load-bearing axioms; Paramasiva is CPT/RAG/GDS corpus governance with Sophia plus Epii co-review and light Anima oversight; Parashakti is graph-relational ML with Sophia lead, Anima cluster/rotation checks, and Pi hot-swap dispatch; Mahamaya is process-reward/federated/genetic pipeline governance with tier-asymmetric review; Nara is Anima-primary with five Anima gates and consent discipline; Epii-on-Epii is recursive governance with anti-self-justification and user-final validation.
- **S0/S0' supplies the current body of evidence.** Agents can read the current public profile, pointer anchor, readiness state, CF/VAK projection, and observability stream through bridge capabilities. They cannot read protected profile internals or rederive private harmonic state.
- **S2 supplies graph-backed context pools.** Agents retrieve bimba, gnosis, etymology, and governed pratibimba handles by namespace through S2/gateway contracts. They do not call external MCP graph surfaces as internal authority and do not collapse Graphiti/Nara memory into canonical topology.
- **S3 supplies durable runtime context.** Agent runs carry session key, DAY/NOW, SpaceTimeDB/gateway readiness, run/event lineage, Graphiti namespace refs, and privacy class. If gateway, SpaceTimeDB, or Graphiti readiness is blocked, the mediation layer declines or degrades visibly.
- **S5 supplies review and autoresearch law.** Every operational output that can affect canon, graph, code, agent config, model artifacts, adapters, or runtime policy becomes a review/autoresearch deposit. Human-required, recursive, deployment, and user-final gates remain non-bypassable, and promotion stays dry-run-only until compiler mutation law is real.
- **M5-3 surfaces are split by depth.** `/body` exposes lightweight review alerts, agent check-in, safe handles, and deep-link intents. `/pratibimba/system` exposes the full Agentic Control Room: VAK route, agent run tree, tool/evidence stream, capacity workflow state, review decision state, dry-run promotion, continuity, and recursive gates.
- **Evidence is the unit of trust.** A mediated run is incomplete unless it records selected artifact URIs, graph refs, source/spec/code/test anchors, profile generation, readiness state, session/DAY/NOW, run id, review id, test output handles, privacy class, and actor/gate lineage.
- **Production tests are real functionality tests.** Acceptable tests call real CLI/gateway/store/bridge paths, use real persisted S5 stores, real S2/S3/S0 payloads or fixtures captured from live upstream services, real Theia/Tauri activation where UI is claimed, and real privacy scans. Mock-only review objects, fake review counts, placeholder panels, and demo agent transcripts do not satisfy readiness.
