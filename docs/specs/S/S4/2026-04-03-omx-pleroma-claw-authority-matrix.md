# OMX Pleroma Claw Authority Matrix

**Date:** 2026-04-03
**Status:** Canonical
**Purpose:** Freeze the semantic authority split for the Pleroma OMX/claw migration so that every
contributor and test can resolve conflicts without guessing.

---

## The Four Authority Layers

| Layer | Name | Role | Source truth |
|-------|------|------|--------------|
| **OMX** | oh-my-codex | Codex-facing workflow and runtime layer | `vendors/oh-my-codex/` + repo-local `.codex/` |
| **ta-onta** | ta-onta specs | Pleroma semantic authority — what Pleroma _means_ and _does_ | `.pi/extensions/ta-onta/pleroma/` specs |
| **PI** | PI native harness | Compatibility substrate — temporary bridge while claw parity is verified | `epi-cli/src/agent/runtime.rs`, `spawn.rs`, `docs/dev/pi-operator-protocol.md` |
| **claw-rust** | claw-rust | Target native harness substrate — long-term replacement for PI once parity gates are green | `vendors/claw-code-parity/`, `epi-cli/src/agent/claw_runtime.rs` (forthcoming) |

---

## What Each Layer Owns

### OMX — Codex Runtime/Workflow Layer

OMX is the **Codex-facing** workflow substrate.  It owns:

- Installed Codex runtime payloads under `.codex/omx-runtime/`
- Repo-local skill and agent discovery directories `.codex/skills/`, `.codex/agents/`
- Project-scoped OMX config (`.omx/setup-scope.json`)
- The workflow scaffolding surface (`omx doctor`, `omx setup`, `omx ralph`, …)
- *Generated* output from `plugins/pleroma/` projection (runtime artefacts, not source)

OMX is **not** the source of truth for Pleroma semantics.  Files produced by `omx init` or `omx
generate` under `.codex/` are runtime output; they are derived, not authoritative.

### ta-onta Specs — Pleroma Semantic Authority

The ta-onta extension specs in `.pi/extensions/ta-onta/pleroma/` and `docs/specs/S/S4/` define
**what Pleroma is**:

- Which skills exist and what they do (`vak-evaluate`, `anima-orchestration`, `vak-coordinate-frame`,
  `day-night-pass`, Technē cluster, Moirai cluster, …)
- Constitutional agent identity and routing tables
- Skill entitlement and ownership
- Day/Night′ topology and Möbius return law
- `plugins/pleroma/` as the canonical **authoring surface** (not `.codex/`)

If OMX and ta-onta disagree, ta-onta specs win.

### PI — Compatibility Substrate (temporary)

The PI native harness remains the **default runtime** while claw parity is verified.  It owns:

- `epi-cli/src/agent/runtime.rs` — operator launch planning
- `epi-cli/src/agent/spawn.rs` — process lifecycle
- `docs/dev/pi-operator-protocol.md` — PI operator contract
- `.pi/extensions/ta-onta/` — compatibility projection of ta-onta semantics into PI runtime

PI is **not** the long-term substrate.  All new capability authoring should target `plugins/pleroma/`
first; PI compat projections are derived.

### claw-rust — Target Native Harness Substrate

claw-rust (`vendors/claw-code-parity/`) is the verified Rust-native replacement for PI.  It will own:

- `epi-cli/src/agent/claw_runtime.rs` — control-plane surface for claw
- `docs/dev/claw-operator-protocol.md` — claw operator contract
- Default runtime for all new `epi agent` operator launches once parity gates are green

claw-rust becomes the **default substrate** in Task 8 of the migration plan, replacing PI as the live
runtime.  Until then it operates as an experimental feature-flagged lane.

## Source Material vs Source of Truth

| Artefact type | Classification | Notes |
|---------------|---------------|-------|
| `plugins/pleroma/skills/*/SKILL.md` | **Source of truth** | Author here; project everywhere else |
| `plugins/pleroma/agents/*/ANIMA.md` | **Source of truth** | Author here |
| `.codex/skills/`, `.codex/agents/` | Derived runtime output | Generated by `epi agent codex install` |
| `.pi/extensions/ta-onta/pleroma/` | Compat projection | Derived from `plugins/pleroma/`; not edited directly |
| `vendors/oh-my-codex/` | Upstream OMX source | Vendored; do not fork skills from here into `plugins/` |
| `vendors/claw-code-parity/` | Upstream claw source | Parity verification harness; not a capability author |
| `docs/specs/S/S4/*.md` | Spec documents | Architecture decisions; drive `plugins/` authoring |

---

## Invariants

1. **OMX ≠ Pleroma authority.** OMX provides runtime plumbing; ta-onta specs provide meaning.
2. **`.codex/` is always generated.** Never hand-edit `.codex/skills/` or `.codex/agents/` as if they
   were source.
3. **PI stays until parity is verified.** Do not remove PI from the default path until claw parity
   gate tests pass (Task 8).
4. **claw-rust is additive, not disruptive.** It runs alongside PI under a feature flag until the
   migration plan's verification gates are green.
5. **`plugins/pleroma/` is the one authoring surface.** New skills, agents, and hooks go here first.
