## VAK Integration Block (Ta Onta extension)

**CF Code:** `(0/1/2)`
**CS State:** Day execution; Night' is retrospective analysis of what the cycle revealed
**CPF Mode:** `(4.0/1-4.4/5)`

# Test-Driven Development with VAK

1. Write one failing real test for the behavior in question.
2. Verify the test fails for the intended reason.
3. Implement the minimum production change to pass.
4. Re-run the targeted test, then broader verification.
5. Capture the resulting VAK block with emphasis on `CT2`, `CP 4.2`, and the active CFP mode.

Night' note: if the cycle reveals hidden ambiguity or a failure pattern, route that to `day-night-pass` instead of pretending Day work is complete.

## Output Format

```text
VAK: [task-short-name]
CPF: (4.0/1-4.4/5)  CT: CT2  CP: 4.2
CF: ((0/1/2)) -> Eros  CFP: CFP[n]  CS: CS[n] / Day
```
