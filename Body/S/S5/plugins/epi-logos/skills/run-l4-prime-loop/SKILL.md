# Run L4-Prime Loop

> `using-epi-logos` runs first. If it hasn't been invoked this turn, go there now.

`L4'` is the scientific verification loop within Square B — the Cause-Experience basin. It gives the objective run-shape for moving from current state to ideal state through verifiable iteration.

Square B companions are always co-present: L1 (what causal structure is being tested), L4 (how the verification process is lived and disclosed — what matters, what is at stake), and L1' (which phenomenal functions orient the evaluation — what is felt before it is measured). The loop runs inside this basin, not in isolation.

The loop: **Observe → Think → Plan → Build → Execute → Verify**, with outer **Learn**.

Build before Execute. This is not negotiable. If you execute without building the evaluative apparatus first, there is nothing honest to verify against — the loop is just motion.

## The Loop

**Observe:** Gather actual context. Current state, user intent, constraints, available evidence, what remains unknown.
*When persistence exists → `/Self/Thought/Questions/` (P0/Truth)*

**Think:** Generate candidate explanations and target states. What are plausible interpretations? What is the desired state? What explains the gap?
*When persistence exists → `/Self/Thought/Traces/` (P1/Mind)*

**Plan:** Design the method. Ordered steps, blockers and risks, why this path rather than another.
*When persistence exists → `/Self/Thought/Challenges/` (P2/Word)*

**Build:** Build the evaluative apparatus before acting — success criteria, validation rules, what pass and fail look like. This is where verification becomes possible rather than theatrical.
*When persistence exists → `/Self/Thought/Patterns/` (P3/Logos)*

**Execute:** Perform the intervention — code changes, research actions, tool runs, structured analysis, writing with operational consequence.
*When persistence exists → `/Self/Thought/Discovery/` (P4/Son)*

**Verify:** Measure the outcome against the criteria established in Build. Pass, fail, or partial pass requiring recursion. If verification fails, recurse honestly rather than decorating failure as insight.
*When persistence exists → `/Self/Thought/Insight/` (P5/Image)*

**Learn:** Carry forward only what survived verification. What is now durable? What method improved? What changes in the next cycle?

## Rules

- Build before Execute. Always.
- Keep paradigm-awareness: evidence and success are never fully frame-neutral.
- Check Square B companions periodically: Is L1 (causal structure) aligned with what you are testing? Is L4 (lived disclosure) registering something the loop is missing? Is L1' (felt evaluation) flagging a qualitative shift the metrics cannot capture?
- If the objective loop is moving but the larger field is still incoherent, hand off to `run-positional-coherence`.
- When the loop surfaces a disclosure/causation gap (what you're building vs what it claims to do), hand the diagonal to `apply-cmea`.
- When the full basin needs mapping rather than just the verification loop, hand to `engage-encounter-axis`.
