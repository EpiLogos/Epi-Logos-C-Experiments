# Compress Thought Artifacts

> `using-epi-logos` runs first. If it hasn't been invoked this turn, go there now.

Compression is reflective, not editorial. Its purpose is to turn process into durable carry-forward without lying about what remains unresolved. Do not leave the field as raw sprawl near compaction. Do not over-resolve paradox to make the summary look clean.

## When to Run

- a task is ending
- a subagent is handing back work
- context compaction is imminent
- a cross-session handoff needs to be durable

## The Sequence

1. Review the active task or subtask — what was actually being pursued?
2. Review which artifact families moved — what contains live material vs noise?
3. Identify what became coherent — what genuinely holds after the run?
4. Identify what remains paradoxical or unresolved — name it precisely, not just implicitly.
5. If the run failed, preserve the failure truthfully and name the next opening it reveals.
6. Write or state the durable synthesis — specific enough that the next cycle can enter cleanly.

## Compression Targets

- Objective work → `/Self/Thought/Insight/` (P5/Image — the (No)Name's synthesis)
- Subjective work → `/Self/Thought/Releasement/` (P5'/Work — Power's synthesis)
- Mixed work → both
- Session-level compression → also copy to `/Self/aham/daily/YYYY-MM-DD/` for durable session logging

## Rules

- Do not delete raw artifacts unless the user explicitly asks
- Do not over-resolve paradox to make the summary neat — if it was not clean, say so
- If persistence exists, write the compression into the target directory
- The compression is done when someone reading it could enter the next cycle without reconstructing context
