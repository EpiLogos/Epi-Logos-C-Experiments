# Choose Modality

> `using-epi-logos` runs first. If it hasn't been invoked this turn, go there now.

Pick the mode that governs the next honest move. This is internal work — determine the mode, then act from it. Do not announce it to the user unless there is a specific reason to.

This orientation is available throughout a run — at the start when direction is unclear, and mid-run when the mode of engagement shifts. It is not a one-time pre-run gate. If a diagnostic run surfaces something that needs building, the modality can shift to applicative. If an applicative run hits a contradiction it cannot resolve by execution, the modality can shift to diagnostic or explorative. Re-invoke whenever the honest mode changes.

A run may braid modes, but it still needs one primary mode driving it and at most one secondary. Multiple modalities at equal weight means the inquiry hasn't been shaped yet.

## The Decision

Choose based on task pressure, not wording style.

- If the task must uncover contradiction or hidden structure → `diagnostic`
- If the task must change state and verify the result → `applicative`
- If the task must open the field before deciding what it is → `explorative`
- If the task must clarify or teach what is already in view → `explanatory`

When genuinely unsure, begin `explorative` and reclassify as soon as the real pressure becomes clear.

## The Four Modes

### Diagnostic

For contradiction, recurring failure, frame conflict, hidden assumptions, anomaly without clear cause, explicit versus implicit fracture.

Default tendencies: `CMEA` primary, `tetralemma` often secondary, `klein` often the honest topology.

### Applicative

For current state → better state, implementation, intervention, experiment, operational change, explicit verification.

Default tendencies: `L4'` scientific loop primary, `torus` common unless contradiction forces richer treatment.

### Explorative

For discovering what the real question is, opening latent structure, resisting premature closure, framing before diagnosis or action.

Default tendencies: QL-heavy question shaping, `torus` or `klein` depending on whether contradiction is present, fast reclassification once the field clarifies.

### Explanatory

For teaching, clarification, comparison, making an architecture or relation intelligible.

Default tendencies: pedagogical conversation — strong pressure against flattening complexity into slogans.

## Wrong Mode Signals

- Explaining when the user actually needs diagnosis
- Diagnosing when the task really needs implementation
- Applying before the question is coherent enough
- Exploring indefinitely to avoid committing to a sharper mode

## Internal Check

Once chosen: name the mode internally and name the task pressure that determined it. Then act from that. If you cannot name the pressure, the mode is probably not settled yet.
