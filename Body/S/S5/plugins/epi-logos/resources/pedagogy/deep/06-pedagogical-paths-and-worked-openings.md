# Pedagogical Paths And Worked Openings

## Purpose

Pedagogy in Epi-Logos should stay conversational. This file gives deep teaching entry paths without hardening the system into a rigid curriculum.

The point is not to define one official lesson order. The point is to give strong openings depending on what the learner is actually struggling with.

## Teaching Rule

Begin from the learner's live pressure.

That pressure usually appears as one of:

- a paradox
- a malformed question
- a fractured field needing diagnosis
- confusion about topology
- confusion about runtime use

Choose the opening that matches that pressure.

## Path 1: Start From Paradox

Use this path when the learner is stuck in contradiction, opposition, or impossibility.

Teach in this order:

1. coherent paradox
2. tetralemma
3. why binary framing is insufficient
4. how `BOTH`, `NEITHER`, and `SILENCE` change the field
5. how return to ground differs from simple resolution

This is often the best first path because it shows the system's distinct value quickly.

## Path 2: Start From Question-Shaping

Use this path when the learner asks vague, flat, or over-binary questions.

Teach in this order:

1. QL as reshaping inquiry before analysis
2. the six positions as question-forms
3. thin-query elicitation
4. implicit vs explicit question structure
5. the move from question-shape to actual traversal

This is often the best practical path for people who want to use the field right away.

## Path 3: Start From Diagnosis

Use this path when the learner wants to understand why a field, design, or worldview is fractured.

Teach in this order:

1. MEF as reflective geometry
2. lenses as complete modes of investigation
3. CMEA as explicit/implicit diagnostic method
4. repression, anomaly, and return of the excluded
5. telos as reintegration rather than critique alone

This is the best path when the value of MEF itself needs to become visible.

## Path 4: Start From Topology

Use this path when the learner needs to understand why the system has this architecture.

Teach in this order:

1. self-reference and genus-1 necessity
2. `4 + 2`
3. Möbius, torus, Klein, lemniscate
4. why recursive return matters
5. how topology becomes runtime

This path is especially useful for theoretically serious learners who want the strongest structural argument first.

## Path 5: Start From Runtime

Use this path when the learner wants to know how the field actually gets used.

Teach in this order:

1. practical coherence walk
2. topological mode selection
3. encounter axis
4. `L4/L4'`
5. `Thought` artifacts as persistence bridge

This is the strongest path for plugin users who need operational clarity before deep doctrine.

## Example Worked Openings

### "I keep hitting contradictions. What is this system actually for?"

Open with:

- coherent paradox
- tetralemma
- the move from contradiction to higher-order field clarity

### "My question feels vague. How do I even start?"

Open with:

- QL modeling
- six positions as question-forms
- one thin-query elicitation pass

### "Why does MEF matter if I already have QL?"

Open with:

- QL as structural rhythm
- MEF as reflective geometry
- CMEA as the differentiator

### "What do the topologies actually change?"

Open with:

- torus as stable circulation
- Klein as doubled encounter
- lemniscate as recursive generation of transcendence

### "How does this become actual agent behavior?"

Open with:

- encounter axis
- practical coherence walk
- artifact persistence and compression

## Beginner, Intermediate, Advanced

### Beginner

Prioritize:

- one key distinction
- one worked opening
- one practical next move

### Intermediate

Prioritize:

- comparison among modes, passes, or squares
- one compact traversal
- one caution against flattening

### Advanced

Prioritize:

- unresolved tensions
- structural derivation
- the psychoid bridge
- topology-to-runtime continuity

## Teaching Cautions

- do not force the learner through one fixed order
- do not treat pedagogy as separate from the live system
- do not flatten runtime into doctrine or doctrine into runtime
- do not overbuild lesson structures where conversational entry would do
